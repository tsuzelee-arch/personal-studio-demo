# Goal Description

The user requested several new features and fixes:
1. Provide an upload option in the Assets Library.
2. Support creating folders in the Assets Library to organize assets.
3. Automatically save images generated by the workflow into a "已完成" (Completed) folder in the Assets Library.
4. Add a mask (遮罩) feature for the workflow to support inpainting/editing.
5. Ensure all parameters take effect when testing with the "Nano Banana 2" model.
6. Fix a bug where typing `@` in the editor causes continuous floating/flickering of the autosuggest menu.

## Proposed Changes

### Assets Panel (index.html, css/style.css, js/assets.js)
- **IndexedDB Update**: Update the `assets` object store format to include a `folder` property (defaults to `'根目錄'` or `'未分類'`).
- **UI Update (`index.html`)**: Add an "上傳資產" (Upload Asset) button and a folder selection sidebar in the Assets panel. Add CSS for the folder list layout.
- **Upload Logic**: Connect the "上傳資產" button to a hidden file input, convert the selected image to Base64, and save it to the current active folder.
- **Folder Logic**: Store the list of custom folders in `localStorage` (e.g., `['根目錄', '已完成']`). Provide a "新增資料夾" (New Folder) button.

### Workflow & Auto-save (js/workflow.js, js/ai-service.js)
- **Mask Node**: Add a new `➕ Mask` button in the workflow toolbar. Create a new Mask node (`type === 'mask'`) that accepts a Base64 mask image upload.
- **Auto-save Logic**: After a successful generation in `workflow.js`, automatically call `AssetsService.saveAsset('Workflow_Output_...', base64Data, '已完成')`. If the "已完成" folder doesn't exist, create it.
- **Nano Banana 2 Parameters**: Update `generateWithNanoBanana2` in `ai-service.js` to properly accept `image` (base), `mask`, `cfg` (temperature), and other parameters from the workflow pipeline, demonstrating that all nodes affect the payload.

### Editor Bug Fix (js/editor.js)
- **Floating Bug**: The autosuggest menu floats continuously because `onEditorInput` re-triggers the menu repositioning on every keypress while in `@` mode. We will fix it so the menu only positions itself when the trigger symbol (`@` or `/`) is first typed, and doesn't update its coordinates if `currentMode` is already active.

## Open Questions

> [!IMPORTANT]
> **Mask Implementation**: Do you want the Mask node to just accept an uploaded black-and-white image file, or do you want a simple drawing tool within the node itself to paint the mask? (Defaulting to an upload input for simplicity).

> [!NOTE]
> **Nano Banana 2 API**: Since `Nano Banana 2` uses the Gemini Image generation API (`gemini-3.1-flash-image`), I will format the payload to include the base image and mask image, though the specific API structure for Gemini image editing (inpainting) might vary. I will implement a standard payload structure to prove the parameters are passed successfully.
